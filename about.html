About this page
This page uses a late-90s server look on purpose. It is plain, direct, and easy to read. The style is meant to make the response feel like something the server returned, not like a modern app.

The point is to help you understand the code path: request comes in, router selects a handler, and the server sends a response back.

How it works
  • The browser sends a request to /.
  • The server reads the request line and headers.
  • The router chooses the matching handler.
  • The response body is returned as HTML.
